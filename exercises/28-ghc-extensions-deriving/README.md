# Exercise 28 — GHC Extensions: Deriving

## Theory

`deriving` in standard Haskell auto-generates instances for a fixed set of
type classes (`Eq`, `Ord`, `Show`, `Read`, `Enum`, `Bounded`).  Several
extensions extend this mechanism dramatically.

### DerivingStrategies

GHC 8.2+ lets you annotate *how* an instance is derived, avoiding ambiguity:

```haskell
{-# LANGUAGE DerivingStrategies #-}

newtype Age = Age Int
  deriving stock   (Show, Eq, Ord)   -- generated by GHC like normal deriving
  deriving newtype (Num)             -- reuse the underlying Int instance
```

There are four strategies:
- **`stock`** — GHC's built-in generator (same as plain `deriving`)
- **`newtype`** — coerce the underlying type's instance
- **`anyclass`** — call `DeriveAnyClass` (requires a default implementation)
- **`via`** — derive via a third type that already has the instance

### GeneralizedNewtypeDeriving

Newtypes wrap exactly one field.  `GeneralizedNewtypeDeriving` lets you
inherit *any* class the underlying type has, not just the stock ones:

```haskell
{-# LANGUAGE GeneralizedNewtypeDeriving #-}

newtype Dollars = Dollars Int
  deriving (Show, Eq, Ord, Num, Enum, Real, Integral)

twentyBucks :: Dollars
twentyBucks = 20   -- numeric literal works!
```

### DerivingVia

`DerivingVia` is the most general strategy.  You specify a *via type* whose
instance is coerced to yours:

```haskell
{-# LANGUAGE DerivingVia #-}

newtype MySum = MySum Int
  deriving (Semigroup, Monoid) via (Sum Int)
```

### DeriveGeneric

`DeriveGeneric` derives `GHC.Generics.Generic`, which many libraries
(aeson, binary, …) use to auto-generate serialisation instances.

### DeriveTraversable

`DeriveTraversable` auto-generates `Functor`, `Foldable`, and `Traversable`
for algebraic data types with a straightforward recursive structure:

```haskell
{-# LANGUAGE DeriveTraversable #-}

data Tree a = Leaf | Node (Tree a) a (Tree a)
  deriving (Functor, Foldable, Traversable)

-- GHC walks the structure automatically:
fmap (+1) (Node Leaf 3 Leaf)   -- Node Leaf 4 Leaf
sum (Node (Node Leaf 1 Leaf) 2 (Node Leaf 3 Leaf))  -- 6
```
